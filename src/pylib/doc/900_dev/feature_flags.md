# motivation / spec
- [DONE] we should have the option to add flags within a file, that can then be passed to the cmdline and fetched in the code
- [DONE] this should not involve any modification in the cli code, instead it should all be doable within a file that imports cli.
- [DONE] since the flags are registered, they should show up in the cli help
- [DONE] they should also show up in the autogenerated CLI docu 
- [DONE] they should also show up in the tui
- [DONE] asking for no specific flag shall return all flags with values, including non-registered values

example:
cli.addflag(name, help, type (determines if flag or value), default)

then, in any function:
cli.getflag(name) returns value

this means flags are not bound to functions, but accessible globally.
flags for specific functions can be added via the usual definition of cmdline options.

# implementation
- the example usage shall be implemented in 100_interactive_docu/000_cli.ipynb
- the functionality shall be implemented within the lib, likely lib/cli, and easily importable outside
